اصطلاحات تحلیل خرابی

روش‌های تحلیل خرابی، ابزارهای ساختاریافته‌ای برای شناسایی، ارزیابی و پیشگیری از خرابی‌های تجهیزات با هدف افزایش قابلیت اطمینان و کاهش هزینه‌های نگهداری هستند.

FMEA
Failure Modes and Effects Analysis
ابزار پیشگیرانه برای شناسایی تمام حالت‌های خرابی بالقوه یک فرآیند یا جزء و پیامدهای آن‌ها
مثال کاربردی:
استفاده از FMEA برای سیستم منبع تغذیه برای ارزیابی ریسک‌ها و راهکارهای کاهش آن
RCA
Root Cause Analysis
رویکردی سیستماتیک و ساختاریافته برای شناسایی و کاهش حالت‌های خرابی بالقوه در یک سیستم، محصول یا فرآیند
ابزارهای رایج:
5 چرا
نمودار استخوان ماهی
درخت علتی
تحلیل پارتو
FRACAS
Failure Reporting, Analysis and Corrective Action System
سیستم حلقه بسته برای ردیابی، بررسی و اصلاح خرابی‌ها به صورت ساختاریافته
فرآیند:
1
گزارش خرابی
2
تحلیل علل ریشه‌ای
3
اقدامات اصلاحی
4
اعتبارسنجی اثربخشی
FTA
Fault Tree Analysis
ابزار گرافیکی برای ترسیم مسیرهای خرابی و علل مشارکت‌کننده در یک رویداد خرابی سطح بالا
مزایا:
ارزیابی کمی احتمال خرابی
شناسایی نقاط ضعف طراحی
اولویت‌بندی اقدامات اصلاحی
نمایش روابط علّی
